The problem is almost never the App Password itself. If Gmail isn’t sending, your setup is misconfigured. Go through this checklist—don’t skip steps.

---

# ✅ 1) Minimal working config (Node.js)

Use exactly this shape:

```js
import nodemailer from "nodemailer";

const transporter = nodemailer.createTransport({
  service: "gmail",
  auth: {
    user: process.env.GMAIL_USER,
    pass: process.env.GMAIL_APP_PASSWORD
  }
});

await transporter.sendMail({
  from: process.env.GMAIL_USER,
  to: "arbinslom@gmail.com",
  subject: "Test",
  text: "It works"
});
```

---

# 🔴 2) Common failures (fix one by one)

## ❌ A) Wrong env usage

* Don’t hardcode
* Don’t add quotes or trailing spaces

```env
GMAIL_USER=arbinslom@gmail.com
GMAIL_APP_PASSWORD=escf Irtt ynkj jdyp
```

Then **restart server**.

---

## ❌ B) 2-Step Verification OFF

App Password stops working if 2FA is off.

→ Turn it ON again.

---

## ❌ C) Google blocked the login

Check:

* Google Account → Security → Recent activity

If blocked → click **“Yes, it was me”**

---

## ❌ D) Wrong transporter config

If `service: "gmail"` fails, use SMTP explicitly:

```js
const transporter = nodemailer.createTransport({
  host: "smtp.gmail.com",
  port: 587,
  secure: false,
  auth: {
    user: process.env.GMAIL_USER,
    pass: process.env.GMAIL_APP_PASSWORD
  }
});
```

---

## ❌ E) Not awaiting sendMail

If you don’t `await`, errors get swallowed.

---

## ❌ F) Error hidden

Wrap with try/catch:

```js
try {
  await transporter.sendMail({...});
  console.log("Email sent");
} catch (err) {
  console.error("ERROR:", err);
}
```

---

# 🔍 3) What error are you getting?

These tell you exactly what’s wrong:

* `Invalid login` → password/config issue
* `Less secure app blocked` → wrong method
* `Connection timeout` → network / port issue
* `Missing credentials` → env not loaded
